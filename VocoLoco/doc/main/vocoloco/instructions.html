Installation Guide

Build from source

Desktop Build

Assuming that you are using linux, open the VocoLoco.pro file in QtCreator, the only problem we can forsee is that the jni.h filepath may not be correct for your system. In the .pro file you can add an include path for the jni on your system. Using linux it is in /usr/lib/jvmjava-1.6.0-openjdk/include. If all else fails you can remove the jni dependancy from the project by removing the #include jni.h from the main.cpp file (this will not affect anything on the desktop build, rather we use the jni when building for android)



Android Build

To build for android, it is very important to modify the httpmanager.h file so that it includes “androidmediaobject.h” instead of “dummydesktopplayer.h”. Also make sure that your jni.h path is correct. This should be set in the .pro file automatically.



Precompiled Locations

Of course you can download the .apk from here

Or run the desktop build by entering the VocoLoco directory and running ./VocoLoco


Misc

Changing Screen Dimensions

If compiling the desktop build, you can change the dimensions of the program by going into the main.qml file and changing the width and height of the first rectangle (line 13). The best thing to do is just modify the height, and leave width alone.
