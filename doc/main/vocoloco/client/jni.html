Java, C++, and the JNI

Running native on the android VM

General View of the JNI

JNI background

The JNI creates a bridge between the JVM and native code e.g C or C++ at the initialization of the JVM, the JNI creates a pointer that contains the interface to the JVM and includes all functions necessary to interact with the JVM and Java objects, including conversitions of data types.

MediaPlayer.cpp

A wrapper of a simple Java class that wraps an instance AndroidMediaPlayer object.

MediaRecorder.cpp

A wrapper of a simple Java class that wraps an instance AndroidMediaRecorder object.

How these C++ classes work

Both C++ classes contain a pointer to the JVM enviorment interface (*env) and basically provide a mapping that directly relates to the actual java code. The method wrappers handle bi-directional data type conversations and message passing to the JVM and or java classes.
